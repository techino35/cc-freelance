## はじめに: なぜ作ったか

フリーランスとして案件を回していると、技術的な実装以外の「管理業務」が意外と重いことに気づきます。

- クライアントへの進捗報告文を毎回ゼロから書く
- 請求書の作成・送付・入金確認を手動で管理
- 案件のパイプラインをスプレッドシートで追いかける
- 新しい案件の見積もりや契約文面を都度考える

これらは1つひとつは小さな作業ですが、積み重なると週に数時間を消費します。エンジニアが本来集中すべき「実装」の時間が、管理業務に食われていく状況です。

そこで構築したのが、Claude Codeを使った「仮想フリーランス会社」です。AIエージェントを部署ごとに分けて、各業務を自動化・半自動化する仕組みを作りました。

---

## 技術構成

システム全体は以下のコンポーネントで構成されています。

### 1. Subagent（部署別エージェント）

各業務領域を担当するエージェントをCLAUDE.mdで定義しています。エージェントはそれぞれ専門的な行動指針を持ち、担当外の業務には関与しません。

### 2. CLAUDE.md（行動指針）

各エージェントの振る舞いはCLAUDE.mdで定義します。「確認せずに進める業務」「必ず確認が必要な業務」を明示することで、エージェントが自律的に動けるようになります。

### 3. cc-freelanceプラグイン

Claude Codeのプラグイン機能を使って、エージェント間の連携ルール（引き継ぎプロトコル）を実装しています。あるエージェントが作業を完了したら、引き継ぎファイルを生成して次の担当エージェントに渡す設計です。

### 4. Google Sheets中央DB

全エージェントが参照・更新する中央データベースとしてGoogle Sheetsを使用しています。GAS Web Appとして公開し、REST APIでアクセスします。

```
GET  ?action=getSummary    → 今月のサマリー
GET  ?action=getPipeline   → 案件パイプライン
GET  ?action=getClients    → クライアント一覧
POST action=addIncome      → 収入追加
POST action=addPipeline    → 案件追加
```

## 各部署の役割と実際の動き

| 部署 | エージェント名 | 主な業務 | 自動化レベル |
|------|--------------|---------|------------|
| 秘書 | secretary-agent | 朝のダッシュボード生成、タスク整理 | 完全自動 |
| 財務 | finance-agent | 請求書作成、入金管理、月次サマリー | 半自動 |
| 営業 | sales-agent | 案件パイプライン管理、提案文作成 | 半自動 |
| クライアント対応 | client-agent | 進捗報告文、納品連絡 | 半自動 |
| エンジニア | engineer-agent | 実装タスク管理、デバッグ記録 | 手動+記録自動 |

**secretary-agent（秘書）**

毎朝、Google Sheetsから当日のタスクと案件状況を取得し、優先順位付きのダッシュボードを生成します。「今日やること」が明確になるため、朝の計画時間がほぼゼロになりました。

**finance-agent（財務）**

納品完了の引き継ぎを受けると、請求書のドラフトを自動生成します。クライアント名、金額、振込先などをGoogle Sheetsから取得して埋め込むため、請求書作成が数分で完了します。

**sales-agent（営業）**

Upworkや直接依頼の案件情報をパイプラインに追加・管理します。案件ステータス（検討中/提案済み/進行中/完了）を追跡し、フォローアップが必要な案件をアラートとして出力します。

**client-agent（クライアント対応）**

エンジニアエージェントから「納品完了」の引き継ぎを受けると、クライアント向けの報告文ドラフトを生成します。過去のやりとりのトーンを参考にしつつ、定型句を避けた自然な文章を出力します。

**engineer-agent（エンジニア）**

実装タスクをMarkdownファイルで管理します。タスクのステータス（open/in-progress/review/done）を更新し、バグ発生時はデバッグログを自動生成します。

## 1日のワークフロー例

### 朝（9:00）

```
secretary-agentを起動
↓
Google Sheetsから当日タスク・案件状況を取得
↓
優先順位付きダッシュボードを出力

【本日のタスク】
1. [高] 岸本さん案件: APIエンドポイント実装（締切: 04/10）
2. [中] 月次レポート確認
3. [低] Upwork新規案件への応募検討
```

### 午前（実装）

```
engineer-agentを起動
↓
タスクファイル確認 → in-progressに更新
↓
実装・デバッグ
↓
完了時にタスクファイルをdoneに更新
↓
引き継ぎファイル生成（→ client-agentへ）
```

### 午後（クライアント対応）

```
client-agentが引き継ぎファイルを受信
↓
進捗報告文のドラフトを生成
↓
人間がレビューして送信

---
[ドラフト例]
岸本様

本日、〇〇機能の実装が完了しました。
テスト環境にて動作確認済みです。
明日中に本番環境へのデプロイ手順をご連絡いたします。
---
```

### 夕方（財務）

```
finance-agentが月次サマリーを確認
↓
未請求案件をリストアップ
↓
請求書ドラフトを自動生成
↓
人間がレビューして送付
```

## 効果: 管理業務が30分以内に収まるようになった

導入前後の比較です。

| 業務 | 導入前 | 導入後 |
|------|--------|--------|
| 朝の計画立て | 20〜30分 | 5分（ダッシュボード確認のみ） |
| クライアント報告文 | 15〜20分/件 | 3〜5分（ドラフトレビューのみ） |
| 請求書作成 | 20〜30分/件 | 5分（ドラフトレビューのみ） |
| 案件パイプライン管理 | 10〜15分/日 | ほぼゼロ（入力のみ） |
| **合計** | **65〜95分/日** | **15〜20分/日** |

管理業務の合計時間が約80%削減されました。浮いた時間を実装や学習に使えるようになったことで、案件の品質も向上しています。

### 副次的な効果

**記録が自動で残る:** タスクファイル、デバッグログ、引き継ぎファイルが自動生成されるため、「あの案件でどんな実装をしたか」を後から振り返れます。

**抜け漏れが減る:** 引き継ぎプロトコルにより、「納品したけど請求書を出し忘れた」「クライアントへの連絡を忘れた」というミスがなくなります。

**判断疲れが減る:** 「これどうすべきか」という小さな判断をエージェントに委ねられるため、重要な意思決定に集中できます。

## まとめ

Claude Codeのエージェント機能を使って、フリーランスの管理業務を大幅に自動化できました。

ポイントをまとめると:

- **部署分け**により、エージェントが専門領域に集中できる
- **CLAUDE.md**で「確認不要な業務」を明示することで自律性が上がる
- **引き継ぎプロトコル**で部署間の連携が途切れない
- **Google Sheetsを中央DB**にすることで全エージェントがデータを共有できる

フリーランスの管理業務は「誰でもできる作業」だからこそ、AIに任せるのが最も費用対効果の高い自動化です。本来集中すべき実装・提案・クライアントリレーションに時間を使えるようになります。
