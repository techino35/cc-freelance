## はじめに

フリーランスエンジニアとして働くと、技術的な仕事と並行して毎月発生する書類仕事があります。請求書の作成、源泉徴収額の計算、インボイス番号の記載、場合によっては業務委託契約書のやりとりと、月末になるたびに同じ作業を繰り返します。

これらをテンプレート化してしまえば、毎月の書類作業を10〜15分に圧縮できます。この記事では、フリーランスエンジニアが実際に使える請求書・契約書のテンプレート構成と、Excelで自動計算させる方法を解説します。

---

## 請求書に必要な項目（インボイス制度対応）

2023年10月から始まったインボイス制度（適格請求書等保存方式）により、請求書に記載すべき項目が増えました。以下がフリーランスエンジニアの請求書に必要な項目です。

### 必須記載事項

| 項目 | 内容 |
|------|------|
| 書類の名称 | 「請求書」と明記 |
| 発行日 | 請求書を発行した日付 |
| 発行者の情報 | 氏名または屋号、住所、連絡先 |
| **適格請求書発行事業者登録番号** | T + 13桁の番号（インボイス登録済みの場合） |
| 取引先の情報 | 請求先の会社名・担当者名 |
| 取引内容 | 業務の内容（具体的に） |
| 取引年月日 | 業務を行った期間 |
| 税抜金額 | 税率ごとに区分して記載 |
| 消費税額 | 適用税率と税額 |
| 合計金額 | 税込の請求金額 |
| 源泉徴収税額 | 差し引く源泉徴収額 |
| 振込先 | 銀行名・支店名・口座種別・口座番号 |
| 振込期限 | 支払期限日 |

インボイス登録をしていない場合、取引先（クライアント企業）は消費税の仕入税額控除を受けられなくなるため、登録を求められるケースが増えています。登録番号は国税庁のインボイス登録申請で取得できます。

## 源泉徴収額の計算方法

デザイン・プログラミング・ライティングなどの業務を個人事業主として受ける場合、報酬から源泉徴収税が差し引かれます。

### 計算式

```
■ 報酬が100万円以下の場合
  源泉徴収税額 = 報酬額（税抜） × 10.21%

■ 報酬が100万円を超える場合
  源泉徴収税額 = (報酬額（税抜） - 1,000,000) × 20.42% + 102,100
```

### 計算例

```
報酬: 300,000円（税抜）の場合
消費税: 300,000 × 10% = 30,000円
消費税込合計: 330,000円

源泉徴収税額: 300,000 × 10.21% = 30,630円
（小数点以下切り捨て）

差引請求金額: 330,000 - 30,630 = 299,370円
```

:::message
源泉徴収の対象になるのは、原則として「個人への業務報酬」です。法人格を持っている（合同会社・株式会社）場合は源泉徴収の対象外となります。
:::

## ExcelまたはGoogleスプレッドシートで自動計算する

毎回手計算するのは非効率です。Googleスプレッドシートで自動計算するテンプレートを作ると、金額を入力するだけで源泉徴収額・消費税・差引請求額が自動で出ます。

### 基本セル構成

```
B2: 報酬額（税抜）  ← ここだけ手入力
B3: 消費税額       = B2 * 0.1
B4: 税込合計       = B2 + B3
B5: 源泉徴収税額   = IF(B2 <= 1000000, INT(B2 * 0.1021), INT((B2 - 1000000) * 0.2042) + 102100)
B6: 差引請求金額   = B4 - B5
```

### GASで請求書PDFを自動生成する

テンプレートシートに数値を入力して請求書PDFを生成するGASスクリプトです。

```javascript
function generateInvoicePDF() {
  var ss = SpreadsheetApp.getActiveSpreadsheet();
  var templateSheet = ss.getSheetByName("請求書テンプレート");
  var ssId = ss.getId();
  var sheetId = templateSheet.getSheetId();

  // PDF生成
  var url = "https://docs.google.com/spreadsheets/d/" + ssId
    + "/export?format=pdf"
    + "&gid=" + sheetId
    + "&portrait=true"
    + "&fitw=true"
    + "&gridlines=false"
    + "&printtitle=false";

  var token = ScriptApp.getOAuthToken();
  var response = UrlFetchApp.fetch(url, {
    headers: { Authorization: "Bearer " + token }
  });

  // ファイル名に請求年月を含める
  var today = new Date();
  var fileName = "請求書_" + Utilities.formatDate(today, "Asia/Tokyo", "yyyyMM") + ".pdf";
  var pdf = response.getBlob().setName(fileName);

  // Google Driveの「請求書」フォルダに保存
  var folderName = "請求書";
  var folders = DriveApp.getFoldersByName(folderName);
  var folder = folders.hasNext() ? folders.next() : DriveApp.createFolder(folderName);
  folder.createFile(pdf);

  Logger.log(fileName + " を保存しました");
}
```

**使い方:**

1. スプレッドシートに「請求書テンプレート」シートを作成
2. GASエディタに上記コードを貼り付け
3. `generateInvoicePDF` を実行するとGoogle Driveの「請求書」フォルダにPDFが保存される
4. 毎月の請求書作成後に実行して保存する運用にする

## 業務委託契約書の必須項目

新規クライアントとの取引では、業務委託契約書を交わすことがリスク管理の基本です。口頭や口約束での合意は後から認識の齟齬が起きやすく、特に報酬未払いや追加作業の範囲でトラブルになります。

### 必ず明記すべき項目

```
1. 業務内容
   - 具体的な成果物・作業範囲を明記する
   - 「Webアプリケーションの開発」ではなく「◯◯機能を持つWebアプリ（仕様書A参照）」と記載

2. 報酬・支払い条件
   - 金額（税抜・税込の区別）
   - 支払い日（例: 請求書受領後30日以内）
   - 振込手数料の負担者

3. 納期・スケジュール
   - 最終納品日
   - 中間マイルストーンがある場合はその日程

4. 知的財産権の帰属
   - 成果物の著作権が誰に帰属するか
   - 納品前・検収前の権利関係

5. 機密保持（NDA）
   - 業務で知り得た情報の取り扱い
   - 有効期間

6. 瑕疵担保責任
   - バグや不具合が発見された場合の対応範囲と期間
   - 「納品後◯ヶ月以内に発見されたバグは無償対応」など

7. 契約解除条件
   - 中途解約時の精算方法
   - どちらから解除できるか・その条件

8. 再委託の可否
   - 外注やサブコントラクターへの再委託を認めるか
```

:::message alert
特に「知的財産権の帰属」と「瑕疵担保の期間・範囲」はトラブルになりやすい箇所です。曖昧なまま進めず、契約書に明記することを強くおすすめします。
:::

## 毎月の書類フロー

実際の月次フローを整理すると以下のようになります。

```
月末 -3日: クライアントに作業完了を連絡
月末 -2日: スプレッドシートに報酬額を入力 → 源泉徴収額を自動計算確認
月末 -1日: GASでPDF生成 → Driveに保存
月末:      PDFをメールに添付して送付
翌月初:    入金確認 → 会計ソフトに記帳
```

新規案件の場合:

```
受注決定時: 業務委託契約書を送付（クラウドサインまたはDocuSign）
契約締結後: 着手
```

## まとめ

- インボイス制度対応には登録番号（T+13桁）の記載が必要
- 源泉徴収は税抜報酬に対して10.21%（100万円超は20.42%）
- スプレッドシートのIF関数で計算を自動化できる
- GASでPDF生成 → Driveへの自動保存まで組める
- 契約書は知的財産権・瑕疵担保の条件を必ず明記する

テンプレートを一度整備してしまえば、毎月の書類仕事は数分で終わります。案件が増えるほど、初期投資として整備する価値が出てきます。
